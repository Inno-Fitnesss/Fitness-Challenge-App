# Withings-интеграция шагов

Работает без своего мобильного приложения: пользователь один раз логинится
через Withings (бесплатное приложение, само умеет считать шаги через
сенсоры телефона — покупать их устройства не нужно), твой бэкенд получает
OAuth2-токен и сам подтягивает шаги из облака Withings.

## 0. Зарегистрируй приложение у Withings

https://developer.withings.com/dashboard/ → создать приложение → получишь
`Client ID` и `Client Secret`. В настройках приложения укажи callback URL:
```
http://ТВОЙ_БЭКЕНД_ХОСТ/me/withings/callback
```
(для локальной разработки — тот же `http://<IP>:8000/me/withings/callback`,
который мы уже использовали)

## 1. Бэкенд — куда класть (пути внутри `backend/`)

| Файл | Путь в проекте |
|---|---|
| `app/db/models/withings.py` | новый файл |
| `app/db/schema/withings.py` | новый файл |
| `app/service/withingsService.py` | новый файл |
| `app/routers/withings.py` | новый файл |
| `main.py` | заменяет твой (уже содержит и steps, и withings роутеры) |
| `app/util/init_db.py` | заменяет твой (уже содержит импорт steps + withings) |

## 2. Добавь в `.env` (backend)

```
WITHINGS_CLIENT_ID=твой_client_id
WITHINGS_CLIENT_SECRET=твой_client_secret
WITHINGS_REDIRECT_URI=http://ТВОЙ_БЭКЕНД_ХОСТ/me/withings/callback
FRONTEND_URL=http://localhost:5173
```

## 3. Фронтенд

| Файл | Путь в проекте |
|---|---|
| `frontend/src/api/withingsApi.ts` | новый файл |
| `frontend/src/components/profile/StepsWidget.tsx` | заменяет твой (добавлена кнопка "Подключить Withings") |
| `frontend/src/pages/ProfilePage.tsx` | заменяет твой (обрабатывает возврат от Withings-логина) |

## Как это работает целиком

1. На странице профиля, если шаги не подключены — кнопка "Подключить Withings".
2. Жмёшь → бэкенд отдаёт ссылку на страницу логина Withings → браузер туда переходит.
3. Логинишься/регистрируешься в Withings (можно только через их веб — без
   установки приложения, если просто хочешь протестировать; для реального
   трекинга шагов пользователю всё же нужно поставить приложение Withings
   на телефон).
4. Withings редиректит обратно на `/me/withings/callback` на бэкенде,
   бэкенд обменивает код на токен и сохраняет его в базе.
5. Бэкенд редиректит браузер обратно на `/profile?withings=connected`.
6. Фронтенд видит этот параметр, сразу дёргает `POST /me/withings/sync`,
   который тянет шаги из Withings и кладёт их в ту же таблицу `steps_daily`.
7. Виджет на странице обновляется — шаги видны.

Ручной повторный синк (шаги за сегодня не приходят сами собой без
переоткрытия страницы) можно повесить на кнопку или на таймер — этого пока
нет, спроси отдельно, если понадобится.
