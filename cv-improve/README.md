# WowFit CV Improve

`cv-improve` — новая версия браузерного CV-прототипа для подсчёта упражнений.

Папка специально сделана плоской:

```text
cv-improve/
  app.js
  index.html
  styles.css
  README.md
```

Старый контрольный образец оставлен отдельно в `cv-test`.

## Версия 2: алгоритмические улучшения (P1–P3)

За опору взята статья *Pūioio: On-device Real-Time Smartphone-Based Automated
Exercise Repetition Counting System* (Sinclair et al., arXiv:2308.02420), где
высокая точность достигается связкой **pose → thresholding → motion (optical
flow) → state machine → counter**. Перенесены три принципа:

- **P1. 3D-углы по worldLandmarks.** Углы суставов (колено, локоть, линия
  корпуса) считаются по метрическим 3D-координатам MediaPipe — это снимает
  зависимость от ракурса камеры. 2D-координаты остаются для наклона к
  горизонту, проверки таза в планке и отрисовки.
- **P2. Подсчёт по развороту движения.** Вместо засчёта по пересечению
  фиксированного порога повтор подтверждается остановкой движения
  (|угловая скорость| < `VELOCITY_EPS`) в верхней и нижней точках, а сам
  повтор — минимальной амплитудой (`MIN_REP_AMPLITUDE`). Пороги `top`/`bottom`
  работают как гистерезис-ограничители. Это убирает дабл-каунты на дрожании
  у порога и пропуски при неполной амплитуде — прямой аналог motion + state
  machine из статьи.
- **P3. Фильтрация сигнала.** Медианный фильтр окна (отбрасывает одиночные
  выбросы landmark'ов) + One-Euro фильтр (адаптивный low-pass: мало лага на
  быстром движении, сильное сглаживание в покое) вместо обычного EMA. One-Euro
  заодно даёт оценку угловой скорости для P2.

### Калибровка на размеченных видео

Пороги пере-тюнены под 3D-углы на видео `test_videos/` (имя файла = число
отжиманий). Офлайн-прогон тем же алгоритмом, что в `app.js`, даёт точное
совпадение:

| Видео | Истина | Счёт |
| --- | ---: | ---: |
| 5.MOV | 5 | 5 |
| 7.MOV | 7 | 7 |
| 17.MOV | 17 | 17 |

Главный вывод калибровки: строгое подтверждение повтора по «нулевой скорости»
(зануление angular velocity в верхней/нижней точке) **теряло быстрые повторы**
при дискретизации 50 мс. Поэтому переход фаз идёт по гистерезис-порогам, а
реальность движения гарантирует проверка амплитуды (`MIN_REP_AMPLITUDE`).
Скорость осталась только для подписи фазы. Тюнинг `top`/`bottom` тоже сильно
ужесточился: камера снимает не идеально сбоку, поэтому `maxTilt` поднят с 25°
до 45°.

Воспроизвести (нужны `pip install mediapipe opencv-python numpy`):

```bash
cd test_videos
python calibrate_pushups.py           # проверка текущих порогов
python calibrate_pushups.py --sweep   # перебор под точное совпадение
```

## Что улучшено для отжиманий

Параметры классических отжиманий подобраны по данным:

```text
data/pushup/labels/correct.npy
data/pushup/labels/incorrect.npy
```

Формат данных:

```text
(количество видео, 150 кадров, 66 координат)
```

Каждый кадр преобразуется в `(33, 2)`, где 33 точки — это MediaPipe landmarks, а координаты — `x, y`.

Главная цель настройки — максимальный recall: все correct-видео должны получить ровно `1` повтор.

## Итоговые параметры pushup (v2, 3D-углы)

```json
{
  "bottom": 110,
  "top": 125,
  "bodyLine": 150,
  "classicMaxTilt": 45,
  "minRepAmplitude": 15,
  "velocityGate": false
}
```

Эти значения перенесены в `app.js` (дефолты слайдеров отжиманий и константы
`PUSHUP_CLASSIC_MAX_TILT` / `MIN_REP_AMPLITUDE`).

> Исторический baseline v1 (пороговая логика на 2D-углах) подбирался по
> `data/pushup/labels/{correct,incorrect}.npy` и давал accuracy 0.85 на
> бинарной задаче «1 повтор / 0». Он заменён на v2 выше.

## Запуск прототипа

Из папки `cv-improve`:

```bash
python -m http.server 8765
```

Открыть:

```text
http://localhost:8765
```

MediaPipe и модель загружаются через интернет, само видео обрабатывается локально в браузере.
